# Исследование рынка и ТОП-10 инструментов для лидогенерации в WhatsApp

Анализ проведен на основе форумов (Reddit, GitHub, n8n community) и документаций API.

## Выбор ТОП-10 решений для твоего проекта

Ниже представлены инструменты, которые лучше всего подходят для связки "Эффективность / Стоимость / Масштабируемость".

| № | Инструмент | Зачем нужен | Почему это ТОП |
| :--- | :--- | :--- | :--- |
| **1** | **Evolution API** | Движок WhatsApp | Лучший Open Source. Нет платы за каждое сообщение. Работает стабильно с группами. |
| **2** | **n8n (Self-hosted)** | Мозг системы | Бесконечные возможности настройки логики. Платишь только за VPS. |
| **3** | **PostgreSQL** | Хранилище (30к+) | Стандарт индустрии. Бесплатно. Легко масштабируется до миллионов записей. |
| **4** | **Gemini 1.5 Flash** | ИИ-аналитик | Самый дешевый и быстрый ИИ на рынке. Идеально для саммари тысяч сообщений. |
| **5** | **pgvector** | Поиск по смыслу | Расширение для Postgres. Позволяет ИИ "помнить" старые диалоги по смыслу, а не по словам. |
| **6** | **Telegram Bot** | Центр уведомлений | Мгновенно шлет алерты тебе в ТГ. Бесплатно и удобно. |
| **7** | **WAHA (WhatsApp HTTP API)** | Альтернатива | Если Evolution API начнет тормозить, WAHA — самый стабильный конкурент на Node.js. |
| **8** | **Typebot** | Квалификация | Если лид перейдет из группы по ссылке, Typebot проведет его по воронке опроса. |
| **9** | **Baserow / Nocodb** | Админка для лидов | Визуальный интерфейс (как в Excel/Airtable) над твоей Postgres базой. Удобно смотреть таблицу. |
| **10** | **Chatwoot** | CRM для ответов | Если ты захочешь отвечать клиентам не из WA, а из одного окна (Omnichannel). |

---

## Обоснование выбора связки (Evolution API + n8n + Postgres)

1. **Экономия:** Ты не платишь Meta за каждое сообщение. Твои расходы — это только аренда VPS ($5-15/мес).
2. **Контроль данных:** 30к сообщений лежат на твоем сервере, а не в облаке стороннего сервиса. Никто не украдет твоих лидов.
3. **Скорость:** Gemini 1.5 Flash обрабатывает текст за миллисекунды, что позволяет мониторить сотни групп "в реальном времени".

## Что говорят на форумах (Reddit/GitHub):
- **Reddit (r/n8n):** Юзеры советуют *всегда* сохранять сырые данные в БД перед отправкой в ИИ, так как ИИ может галлюцинировать или тупить.
- **GitHub:** В репозиториях Evolution API рекомендуют использовать Redis для кэширования сессий, чтобы "не отлетала" авторизация при большом потоке сообщений.

## Мой выбор №1 для тебя:
**PostgreSQL + pgvector**. Это позволит тебе в будущем сделать запрос: *"Кто из моих контактов за последний месяц интересовался перевозкой зерна из КЗ в РФ?"* и получить точный список за 1 секунду.
